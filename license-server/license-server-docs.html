🔐 License Server Documentation

Central Envato License Verification — Multi-Product Edition

PHP 7.4+ MySQL 5.7+ Multi-Product Domain Locked Envato API v3

Architecture Overview

The license server acts as a proxy between buyer servers and the Envato API. Your Envato Personal Token is stored exclusively on your server — it is never distributed in buyer files.

Buyerruns install.php
→
verify_license.phpon buyer's server
→
License Serveryour hosting
→
Envato APIapi.envato.com

Key Security Properties

  • Envato token is never in buyer files — lives only on your server
  • Each product has its own unique secret key — one compromised product doesn't affect others
  • Activations are domain-locked — one purchase code works on one domain only
  • All buyer requests are authenticated with a product secret before the license server responds

How It Works (Full Request Flow)

Step-by-Step: First Activation

1

Buyer opens install.php

Step 1: buyer enters DB credentials → config files written automatically.
Step 2: buyer enters Envato purchase code.

2

JS fetches api/verify_license.php

A POST request is sent with purchase_code to the buyer's server.

3

verify_license.php calls your license server

Sends: product_id, product_key (shared secret), purchase_code, domain.

4

License server validates the product secret

Looks up product_id in the PRODUCTS[] config. Compares product_key. Rejects unknown products or wrong keys.

5

License server queries Envato API

Calls api.envato.com/v3/market/buyer/purchase?code=... with your Envato token.

6

On success: domain lock + DB record

Stores (product_id, purchase_code, buyer_username, domain) in your DB. Unique key prevents the same code activating on a second domain for the same product.

7

License file written on buyer's server

server/config/license.php is written with SN_LICENSED = true. This file is checked on every Flutter API request via license_check.php.

8

Subsequent API requests check local license file

api/settings/get_settings.php includes license_check.php — checks config/license.php exists and is valid. No network call on every request.

Files Overview

Files are split into two categories: Seller-side (runs on YOUR server, never distributed) and Buyer-side (distributed in the product ZIP).

Seller-Side Files (your server only — never distribute)

license-server/index.php Seller
Main API endpoint. Handles all verification requests. Validates product secrets, calls Envato API, enforces domain lock.
license-server/admin.php Seller
Admin dashboard. View all activations per product, revoke licenses, see activity logs.
license-server/config/settings.php Seller
Your Envato token + PRODUCTS[] array. One entry per product. Add new products here.
license-server/config/database.php Seller
License server DB credentials (separate DB from buyer's app).
license-server/setup.sql Seller
Import once to create the license DB tables: activations + activation_logs.
license-server/.htaccess Seller
Routes all requests to index.php. Blocks direct access to config/ and cache/. Basic Auth on admin.php.

Buyer-Side Files (distributed in product ZIP)

server/install.php Buyer
3-step setup wizard. Writes DB config files (Step 1) and activates license (Step 2). Redirected to admin on Step 3.
server/api/verify_license.php Buyer
Called by install.php. POSTs to your license server with product_id + product_key. Writes config/license.php on success.
server/api/config/license_config.php Buyer
Per-product config. Contains LICENSE_SERVER_URL, PRODUCT_ID, PRODUCT_SECRET (no Envato token). You fill this before distributing.
server/api/config/license_check.php Buyer
Guard middleware. Include at top of any API endpoint. Returns 403 if config/license.php doesn't exist or is invalid.
server/admin/login.php Buyer
Has license gate at top — redirects to install.php if not licensed. Admin panel is inaccessible without activation.

1. Deploy the License Server

The license-server/ folder is deployed on your own hosting. It handles ALL your products from one URL.

  1. Upload license-server/ to your hosting (e.g. public_html/license/)
  2. Set folder permissions: 755 for directories, 644 for PHP files
  3. Create the cache/ directory with write permission (chmod 775 cache/)
  4. Ensure mod_rewrite is enabled (Apache) — .htaccess is included
  5. Confirm it runs at: https://license.yourdomain.com/
⚠️ Use HTTPS only. Requests from buyer servers carry the product secret — always use SSL.

Test the Endpoint

curl -X POST https://license.yourdomain.com/ \
  -d "action=check&product_id=shortnews&product_key=YOUR_SECRET&purchase_code=TEST&domain=test.com"

# Expected (if secret is wrong):
{"success":false,"message":"Unauthorized request."}

2. Database Setup

The license server uses its own separate MySQL database, isolated from any buyer's app database.

-- Create the database on your server
CREATE DATABASE shortnews_licenses CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

-- Then import the schema
-- (Run from CLI or phpMyAdmin)
mysql -u your_user -p shortnews_licenses < license-server/setup.sql

Edit license-server/config/database.php:

define('LS_DB_HOST', 'localhost');
define('LS_DB_USER', 'your_db_user');
define('LS_DB_PASS', 'your_db_password');
define('LS_DB_NAME', 'shortnews_licenses');

3. Envato Personal Token

One Envato token verifies purchase codes for all your products.

How to Get Your Token

  1. Go to build.envato.com
  2. Click "Create a new token"
  3. Enable permission: "View the user's purchases of the app creator"
  4. Copy the generated token

Set it in license-server/config/settings.php:

define('ENVATO_TOKEN', 'your_token_here'); // 40-char string
🚨 Never commit settings.php to version control. It is already listed in .gitignore.

4. Product Secrets

Each product has its own secret key in the PRODUCTS[] array. This secret is shared between your license server and the buyer's license_config.php.

// license-server/config/settings.php
define('PRODUCTS', [
    'shortnews' => [
        'name'   => 'ShortNews — Flutter News App',
        'secret' => 'abc123XYZ...', // min 32 random chars
    ],
    'jobapp' => [
        'name'   => 'JobApp — Flutter Job Board',
        'secret' => 'def456UVW...', // different secret!
    ],
]);
ℹ️ Generate strong secrets at random.org/strings — use Letters + Digits, min 32 characters.

5. Protect the Admin Dashboard

The admin dashboard is protected with HTTP Basic Auth. Set it up once:

# SSH into your server and run:
htpasswd -c /home/your_user/.htpasswd admin
# Enter a strong password when prompted

# Then edit license-server/.htaccess:
# Change this line:
AuthUserFile /path/to/.htpasswd
# To the actual path, e.g.:
AuthUserFile /home/your_user/.htpasswd

Access at: https://license.yourdomain.com/admin.php

Adding a New Product

When you publish a new product on CodeCanyon, you need to make 4 changes — 1 on your server and 3 in the new product's distributed files.

New Product — Server-Side Change

Edit license-server/config/settings.php and add one entry to the PRODUCTS[] array:

define('PRODUCTS', [
    'shortnews' => [ ... ],       // ← existing product

    // ── ADD NEW PRODUCT HERE ──────────────────────
    'your_new_product_id' => [
        'name'   => 'Your Product Name',
        'secret' => 'GENERATE_NEW_UNIQUE_SECRET_MIN_32_CHARS',
    ],
    // ─────────────────────────────────────────────
]);
✅ That's the only server-side change needed. No DB migration, no redeployment — just add the entry and save.

New Product — Buyer-Side Files

Every new product needs these 4 buyer-side files configured and distributed in its ZIP.

File 1 — api/config/license_config.php

Change PRODUCT_ID and PRODUCT_SECRET to match the new product's settings.php entry:

<?php
define('LICENSE_SERVER_URL', 'https://license.yourdomain.com/');
define('PRODUCT_ID',         'your_new_product_id');  // ← MUST match settings.php key
define('PRODUCT_SECRET',     'GENERATE_NEW_UNIQUE_SECRET_MIN_32_CHARS'); // ← MUST match settings.php secret

File 2 — api/verify_license.php

This file is identical for all products — copy as-is from ShortNews. No changes needed. It reads PRODUCT_ID and PRODUCT_SECRET from license_config.php dynamically.

// verify_license.php — same file for all products, no changes needed
require_once '../config/license_config.php'; // reads the product-specific values

File 3 — api/config/license_check.php

This file is also identical for all products — copy as-is. It only checks the local config/license.php file, which is product-agnostic.

File 4 — install.php

Copy as-is, but update the product name in the UI and the confirmation text for buyers:

<!-- Change only the display name in the brand section: -->
<h1>YourProductName</h1>   <!-- was: ShortNews -->
<p>Installation Wizard</p>

File 5 — admin/login.php (or any admin entry point)

Add the license gate at the very top — identical for all products:

<?php
// License gate — identical for all products
$_lf = __DIR__ . '/../config/license.php';
if (!file_exists($_lf)) { header('Location: ../install.php'); exit; }
require_once $_lf;
if (!defined('SN_LICENSED') || SN_LICENSED !== true) { header('Location: ../install.php'); exit; }
unset($_lf);
// ── rest of admin/login.php below... ──
ℹ️ Also add require_once '../config/license_check.php'; to any API endpoint that should be protected (e.g. your equivalent of get_settings.php).

Using the Admin Dashboard

Access at: https://license.yourdomain.com/admin.php

Features

  • Product filter tabs — view all activations or filter by specific product
  • Per-product stat cards — total + today's activation count for each product at a glance
  • Activations table — full list with product badge, purchase code, buyer username, domain, date
  • Activity log — last 20 events with timestamps and IP addresses

Revoking Licenses

Use the Revoke button in the admin dashboard to remove an activation. This is useful for:

  • Buyer requested a refund (Envato handles payment, you handle access)
  • Suspected abuse or license sharing
  • Buyer wants to move to a new domain (revoke + re-activate)
⚠️ Revoking only removes the DB record on YOUR server. The buyer's local config/license.php still exists. To immediately cut access, you must also enable Periodic Re-Validation.

Periodic Re-Validation

By default, after initial activation the buyer's config/license.php file is checked locally — no network call per API request. This is fast but means revoked licenses still work until the local file is deleted.

To enable 7-day online re-checks, uncomment the block at the bottom of server/api/config/license_check.php:

// Uncomment this entire block in license_check.php:
if (defined('SN_PURCHASE_CODE')) {
    $checkFile = sys_get_temp_dir() . '/sn_lcheck_' . md5(SN_PURCHASE_CODE) . '.txt';
    $lastCheck = file_exists($checkFile) ? (int) file_get_contents($checkFile) : 0;
    if (time() - $lastCheck > 7 * 86400) {
        // ... pings your license server's 'status' action
        // If server returns active=false → deletes local license.php → returns 403
    }
}

License Server API Reference

All requests: POST https://license.yourdomain.com/

Action: check — Verify & Activate

ParameterRequiredDescription
actionYesMust be check
product_idYese.g. shortnews
product_keyYesProduct's secret from settings.php
purchase_codeYesEnvato purchase code from buyer
domainYesBuyer's domain (set from $_SERVER['HTTP_HOST'])
 
// Success response:
{"success": true, "message": "License activated successfully!", "buyer": "buyer_username"}

// Error — wrong secret:
{"success": false, "message": "Unauthorized request."}

// Error — domain mismatch:
{"success": false, "message": "This purchase code is already activated on: otherdomain.com. One license = one domain."}

// Error — invalid code:
{"success": false, "message": "Invalid purchase code. Please check and try again."}

Action: status — Check Existing Activation

ParameterRequiredDescription
actionYesMust be status
product_idYese.g. shortnews
product_keyYesProduct's secret
purchase_codeYesThe purchase code to check
domainYesDomain to verify against
 
// Active:
{"success": true, "active": true, "buyer": "username", "activated": "2024-03-25 10:00:00"}

// Not active / revoked:
{"success": true, "active": false}

Troubleshooting

❌ "Unauthorized request." on every activation

The PRODUCT_SECRET in the buyer's license_config.php doesn't match the secret in your settings.php for that product_id. Check both files — they must be character-for-character identical.

❌ "Unknown product ID."

The PRODUCT_ID in license_config.php doesn't match any key in your PRODUCTS[] array. Check spelling and case — keys are case-sensitive.

❌ "Could not reach the license server."

The buyer's server couldn't connect to your license server URL. Check: (1) your server is online, (2) LICENSE_SERVER_URL has a trailing slash, (3) HTTPS is valid, (4) buyer's PHP has curl enabled.

❌ install.php shows blank page after DB step

Usually a PHP error in the config file write. Check that server/api/config/ and server/admin/config/ are writable (chmod 755).

❌ License activated but admin panel redirects to install.php

The license file wasn't written. Check that server/config/ directory exists and is writable. The directory is created automatically but may fail due to permissions.

❌ Database "already activated" error but buyer can't access admin

The activation record exists in the DB but config/license.php is missing on the buyer's server. Have them go to install.php again — it will detect the existing activation and re-write the license file.
